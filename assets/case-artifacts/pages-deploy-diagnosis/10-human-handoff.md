# 03 GitHub Pages 部署失败诊断 - 人工交接清单

## 中文

- 只把新 run 作为成功依据，不回头修改旧失败 run 的判断。
- 发布修复和内容改动尽量分开，便于回滚。
- 宣布完成前必须同时确认 Actions 与线上地址。

## English

- Use only the new run as success evidence; do not reinterpret the old failed run.
- Keep publish fixes and content edits separate where possible for rollback.
- Before declaring completion, confirm both Actions and the live URL.
