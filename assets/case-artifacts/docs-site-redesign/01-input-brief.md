# 04 本地文档站批量改版

## 中文

### 任务摘要
把静态文档站一次性改成深色产品工作台，并用页面数量、链接、截图和禁用词检查证明未破坏发布链路。

### 可读取材料
- 改版前锁定页面数量、目录、发布命令和本地服务地址。
- 视觉目标拆成颜色、密度、导航、表格、代码块和移动端规则。
- 截图只展示本站内容，避免混入无关品牌和业务数据。

### 输入样例

```text
改版约束
底色 #141414，面板 #1e1e1e / #242424，边框 #333，强调色 #ff922c
页面数量保持 44
README 需要展示新版截图
本地路径和线上路径都要可访问
```

## English

### Task Summary
Redesign a static documentation site into a dark product workspace and prove the publish chain remains intact with page count, links, screenshots, and blocked-word checks.

### Readable Materials
- Before redesign, lock page count, folders, deploy commands, and local service URLs.
- Break the visual target into color, density, navigation, tables, code blocks, and mobile rules.
- Screenshots show only this site and do not include unrelated brand or business data.

### Input Sample

```text
Redesign constraints
Background #141414, panels #1e1e1e / #242424, border #333, accent #ff922c
Keep 44 pages
README must show updated screenshots
Local and live paths must be accessible
```
