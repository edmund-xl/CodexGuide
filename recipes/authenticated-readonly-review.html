08 登录态网页只读检查

在用户已登录的浏览器里只读检查页面状态，记录可见问题、禁点清单和待用户确认动作。

文档状态
发布草案 v0.2
最后核对
2026-05-28
适用读者
账号管理者、运营、客服负责人
预计耗时
25-45 分钟
风险等级
中到高
入口 用户浏览器 + 只读操作
材料 已登录页面、检查目标、禁点清单、截图脱敏规则
证据 可见状态摘要、脱敏截图、禁点清单、待确认动作
交付物 readonly-review.md、safe-screenshots/、confirmations.md
风险 中到高
预计耗时 25-45 分钟

任务背景

在用户已登录的浏览器里只读检查页面状态，记录可见问题、禁点清单和待用户确认动作。

账号页面检查 账号管理者、运营、客服负责人 中到高

输入材料

本次任务先锁定可读取材料、禁止动作和输出格式，再开始生成或检查。

  • 用户已经在自己的浏览器登录，任务不读取密码、Cookie 或本地会话文件。
  • 先列出禁止动作：提交、购买、授权、删除、转账、邀请、公开发布。
  • 截图前隐藏不需要展示的邮箱、余额、账号 ID 和私人资料。
只读检查目标
页面：已登录的账单概览
想确认：是否有失败付款、是否有异常提示、发票入口是否可见
禁止：更改计划、提交付款、删除成员、发送邀请、导出完整数据

运行环境

  • 只采集屏幕可见信息，不导出账号数据。
  • 可点击范围限制为导航、筛选、展开详情和关闭弹窗。
  • 所有会改变账户状态的动作写入 confirmations.md，由用户处理。

每个案例都提供一组可打开的演示文件：输入任务单、证据表、结果片段、验收 runbook、执行转录、交付预览、前后对比、质量评分和证据看板。读者可以先看材料，再替换成自己的任务。

操作剧本

  1. 先确认当前页面标题、用户目标和禁点清单。
  2. 读取页面状态和可见提示，不点击会改变账户、余额、权限或公开内容的控件。
  3. 把需要用户处理的动作列为待确认，不替用户执行。

Read page state in the user's signed-in browser, recording visible issues, no-click controls, and actions needing user confirmation.

Task Background

Read page state in the user's signed-in browser, recording visible issues, no-click controls, and actions needing user confirmation.

Authenticated page review Account owners, operators, and support leads Medium to high

Input Materials

This task first locks readable materials, prohibited actions, and output format before generation or inspection.

  • The user is already signed in; the task does not read passwords, cookies, or local session files.
  • List prohibited actions first: submit, purchase, authorize, delete, transfer, invite, and publish.
  • Before screenshots, hide unnecessary email, balance, account IDs, and private profile details.
Read-only review goal
Page: signed-in billing overview
Need to confirm: failed payments, warning banners, invoice entry visibility
Prohibited: change plan, submit payment, delete members, send invites, export full data

Run Environment

  • Capture only visible screen information and do not export account data.
  • Clickable scope is limited to navigation, filters, expanding details, and closing dialogs.
  • All actions that change account state go into confirmations.md for the user.

  2. Read page state and visible notices without clicking controls that change account, balance, permissions, or public content.
  3. List actions needing user handling as confirmations instead of executing them.
