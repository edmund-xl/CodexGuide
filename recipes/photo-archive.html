05 Codex × Local Files：整理照片命名与索引

把活动照片按日期、场景和用途生成可检索目录。

文档状态
发布草案 v0.2
最后核对
2026-05-28
适用读者
业务人员、创作者、项目维护者
预计耗时
30-45 分钟
风险等级
低到中

业务目标

把活动照片按日期、场景和用途生成可检索目录。

  • 领域：文件整理。
  • 目标是生成可人工审阅的中间交付物，而不是自动对外发布。
  • 所有外部发送、公开发布或账号操作都必须由用户完成。

输入材料

输入材料必须最小化、可追溯、可删除。不要把完成任务不需要的私人信息交给 Codex。

  • 任务说明：目标、受众、输出格式和限制条件。
  • 任务材料：只包含完成案例所需的文件、链接或摘录。
  • 验收清单：事实、格式、风险和缺口。

执行流程

让 Codex 先做结构化分析，再生成草稿。不要让它跳过材料核对或不确定事项标注。

  • 用一句话说明案例：“05 Codex × Local Files：整理照片命名与索引”。
  • 要求 Codex 先列出材料清单和处理步骤。
  • 生成第一版输出后，要求列出不确定项、缺失材料和人工确认点。
  • 按验收清单修改，而不是重复生成。

提示词合同

提示词应像工作单一样清晰，明确 Codex 不能越权补充事实。

  • 请先说明你会读取哪些材料，以及每份材料的用途。
  • 不要补充输入材料之外的事实；缺失内容请标注为待确认。
  • 输出必须包含正文、检查清单和下一步建议。

交付物

案例交付物应可被另一个人独立检查。

  • 一份结构化草稿或变更清单。
  • 一份不确定项和待补材料列表。
  • 一份人工验收清单。

验收标准

案例完成不等于内容完美，而是达到可审阅、可修改、可追溯的标准。

  • 所有关键事实都能回到输入材料。
  • 输出没有未经确认的承诺、金额、日期或账号动作。
  • 读者能根据交付物继续人工编辑或发布。
  • 风险和限制被显式列出。

文档质量验收标准

本页只有在读者能按步骤执行、识别风险、产出交付物并完成验收时才算达标。

  • 中文和英文信息一致，没有遗漏关键限制。
  • 读者能根据本页完成一个可回退的低风险任务。
  • 任务结束后有输出、证据和待确认事项。
  • 涉及动态事实时，读者能回到官方文档复核。

05 Codex × Local Files: Organize photo names and index

Create a searchable archive by date, scene, and usage.

Status
Release draft v0.2
Last verified
2026-05-28
Audience
Operators, creators, and project maintainers
Estimated time
30-45 minutes
Risk level
Low to medium

Business objective

Create a searchable archive by date, scene, and usage.

  • Domain: File organization.
  • The goal is to produce a human-reviewable intermediate deliverable, not to publish automatically.
  • All external sending, public publishing, or account actions must be performed by the user.

Input materials

Input materials must be minimized, traceable, and removable. Do not provide private information that is not required for the task.

  • Task brief: objective, audience, output format, and constraints.
  • Task material: only files, links, or excerpts needed for the recipe.
  • Acceptance checklist: facts, format, risks, and gaps.

Workflow

Ask Codex to perform structured analysis before drafting. Do not let it skip material checks or uncertainty labels.

  • Describe the recipe in one sentence: "05 Codex × Local Files: Organize photo names and index".
  • Ask Codex to list materials and processing steps first.
  • After the first output, ask for uncertainties, missing materials, and human-confirmation points.
  • Revise against the checklist instead of regenerating repeatedly.

Prompt contract

The prompt should read like a work order and explicitly prohibit Codex from inventing unsupported facts.

  • First state which materials you will read and how each will be used.
  • Do not add facts outside the provided inputs; mark missing content as to be confirmed.
  • The output must include the body, a review checklist, and recommended next steps.

Deliverables

Recipe deliverables should be independently reviewable by another person.

  • A structured draft or change list.
  • A list of uncertainties and missing materials.
  • A manual acceptance checklist.

Acceptance criteria

A recipe is complete when it is reviewable, editable, and traceable, not when it looks perfect.

  • All key facts can be traced back to the provided inputs.
  • The output contains no unconfirmed promises, amounts, dates, or account actions.
  • A human can continue editing or publishing from the deliverable.
  • Risks and limitations are explicitly listed.

Documentation Quality Acceptance Criteria

This page meets the documentation standard only when readers can follow the procedure, identify risks, produce deliverables, and complete acceptance checks.

  • Chinese and English content are equivalent and do not omit critical constraints.
  • A reader can complete a reversible low-risk task using this page.
  • The task ends with output, evidence, and confirmation items.
  • For dynamic facts, the reader can verify against official documentation.
